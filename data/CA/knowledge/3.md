## Document Overview

**Subject:** Computer Science

**Summary:** The Thumb instruction set is a 16-bit subset of the 32-bit ARM instruction set, designed to improve performance on systems with 16-bit data buses. It encodes a subset of ARM instructions, offering higher performance than ARM on 16-bit buses, but lower performance on 32-bit buses. This makes it suitable for memory-constrained systems. The Thumb instruction set is a subset of the ARM instruction set, and its decoding process is discussed in detail. The chapter also discusses the register usage and load-store instructions in the Thumb instruction set, including MOV, MUL, MVN, and more. Additionally, the chapter covers ARM-Thumb interworking, which links ARM and Thumb code together for assembly and C/C++ programming. The document also discusses data processing instructions, which perform arithmetic and logical operations on register values, and load-store instructions, which transfer data between memory and registers. Furthermore, the chapter covers multiple-register load-store and stack instructions, including POP and PUSH, which are used to manipulate registers on the stack. Finally, the chapter discusses the Thumb SWI instruction, which executes a software interrupt.

> **Summary:** This slide provides a summary of key features of the Thumb instruction set, including its 16-bit length, improved code density, ARM-Thumb interworking, conditional execution for branch instructions, and limitations regarding multiple-register load-store, stack instructions, and access to coprocessors or status registers.

Here is the cleaned text:

Summary

• All Thumb instructions are 16 bits in length.
• Thumb provides approximately 30% better code density over ARM code in a high-level language such as C and C++.
• ATPCS defines how ARM and Thumb code call each other, called ARM-Thumb interworking.
• In Thumb, only the branch instructions are conditionally executed. The barrel shift operations (ASR, LSL, LSR, and ROR) are separate instructions.
• The multiple-register load-store instructions only support the increment after (IA) addressing mode.
• The Thumb instruction set includes POP and PUSH instructions only support a full descending stack.
• There are no Thumb instructions to access the coprocessors, cpsr, and spsr.
